## Overview

The Material Tracking System is a mobile application used to monitor the process of material delivery, from the supplier all the way to goods receipt at the factory. The system supports the full delivery lifecycle, starting from Purchase Order (PO) issuance, delivery scheduling, Delivery Order (DO) status monitoring, up to real-time goods receipt verification through barcode scanning. As an intern at PT Greatsoft Solusi Indonesia, I was placed on the mobile development team to build specific modules of this application using Flutter, working alongside one senior developer.

## Key Features

- **Purchase Order (PO) Module**: PO list and detail pages, approve, reject, and negotiate schedule actions, data filtering with lazy loading, status adjustments, and document preview and download.
- **Delivery Order (DO) Module**: DO creation, list and detail pages, a delivery timeline view, inspection result reporting, DO deletion, barcode scanning, and delivery status adjustments.
- **Dashboard and Notifications**: A summary dashboard for PO and DO statuses, a lazy-loaded notification system, and automatic dashboard data refresh.
- **Account and Profile Management**: Login and logout flows, profile editing, and password and profile photo updates.
- **UI/UX and Reusable Components**: A set of reusable UI components, bottom navigation and interactive dialogs, and supporting components such as filters and date pickers, all built to keep the app's views consistent.

## My Role and Contributions

I worked as a Mobile Frontend Developer alongside one senior developer, jointly building the application's frontend using Flutter and Dart. My contributions covered the full PO and DO modules described above, along with lazy-loaded notifications and a library of reusable UI components used to maintain visual and functional consistency across the app.

## Challenges and Learnings

This project required adapting to Flutter, a stack I was not yet experienced with, directly within a live production environment for a real client. This experience taught me that professional work demands faster and more independent learning, since there was little room for a long ramp-up period before contributing meaningful, production-ready code.

## Outcome

The project contributed directly to accelerating the application's development timeline and improving the quality of deliverables that PT Greatsoft Solusi Indonesia provided to its client, supporting the factory's daily material delivery monitoring process.
